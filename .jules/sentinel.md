## 2025-02-28 - Hardcoded Secret in authController.ts
**Vulnerability:** A hardcoded fallback secret was used in `server/controllers/authController.ts` and `server/middlewares/authMiddleware.ts` for signing and verifying JSON Web Tokens (JWT). If the `JWT_SECRET` environment variable wasn't set, the system fell back to `'fallback_secret_do_not_use_in_prod'`, allowing attackers who know or discover this constant to forge valid tokens and gain unauthorized access.
**Learning:** Hardcoded fallback secrets in production code present a significant security risk, as they effectively bypass authentication if environment variables are accidentally omitted or misconfigured. Applications should fail securely (e.g., throwing an error on startup) rather than degrading to an insecure state.
**Prevention:** Always assert the presence of critical security configuration (like secrets) at runtime or module load time. Use throw statements to halt the application if required secrets are missing, ensuring it's impossible to run with known insecure defaults.
